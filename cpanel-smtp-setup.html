📧 ALIVIO247 cPanel SMTP Setup Guide

Complete setup instructions for using cPanel email with your contact form.

🔧 Step 1: Create Email Account in cPanel

  1. Log into your cPanel
  2. Go to "Email Accounts"
  3. Create or verify the email account: support@alivio247.com
  4. Set a strong password and note it down
  5. Make sure the email account is active

⚙️ Step 2: Update SMTP Settings

Edit the file contact-handler-smtp.php and update these settings:

$smtpConfig = [ 'host' => 'mail.alivio247.com', // Change to mail.yourdomain.com 'port' => 587, // Keep as 587 for TLS 'encryption' => 'tls', // Keep as tls 'username' => 'support@alivio247.com', // Your cPanel email 'password' => 'YOUR_ACTUAL_PASSWORD', // Password you set in Step 1 'from_email' => 'support@alivio247.com', 'from_name' => 'ALIVIO247 Contact Form' ];

🚨 Important Settings to Update:

  • host: Usually mail.yourdomain.com (replace yourdomain.com with your actual domain)
  • password: Replace YOUR_ACTUAL_PASSWORD with the real password from Step 1
  • username: Must match the email account you created in cPanel

🔍 Step 3: Find Your Mail Server Settings

In cPanel, go to "Email Accounts" and click "Setup Mail Client" next to your support@alivio247.com account.

Look for these settings:

  • Incoming Server: This is usually your host
  • Outgoing Server: This is usually the same as incoming
  • Port: Usually 587 for SMTP with TLS
  • Security: Usually TLS

📝 Step 4: Common cPanel Mail Server Formats

Your mail server host is typically one of these formats:

mail.yourdomain.com yourdomain.com servername.yourhostingprovider.com

Replace yourdomain.com with your actual domain name.

🧪 Step 5: Test the Setup

  1. Save your changes to contact-handler-smtp.php
  2. Upload the file to your website
  3. Visit your website's contact form
  4. Submit a test message with your own email
  5. Check both email addresses for received messages

✅ Expected Results:

  • You receive a "Thank You" email
  • support@alivio247.com receives a notification
  • alivio247alivio@gmail.com receives a notification
  • Success message appears on the contact form

⚠️ Troubleshooting Tips:

  • Check spam folders - new emails might be filtered
  • Verify domain DNS - make sure MX records are set up
  • Contact hosting support - if you can't find mail server settings
  • Try different ports: 25, 465 (SSL), or 587 (TLS)
  • Check cPanel error logs - for detailed error messages

🆘 Still Having Issues?

If emails still don't send, the contact form will:

  • Save customer messages to backup files
  • Show detailed error messages
  • Prevent losing any customer inquiries

You can manually respond to customers using the saved data.
